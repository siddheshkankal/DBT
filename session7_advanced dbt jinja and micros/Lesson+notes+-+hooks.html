Lesson notes - hooks

Hooks are useful for when you want to run some SQL before or after a whole dbt run, or before or after a specific dbt model.

I’d recommend reading more about the use cases (link), but essentially if you want to run a hook:

  • Before / after a model, you’ll need to use your {{ config () }} block like this:
    {{ config(
    post_hook=[
      "alter table {{ this }} ..."
    ]
) }}
  • Before / after a whole dbt run / seed / snapshot, you’ll need to set it up in your dbt_project.yml (like we do in this lesson) like this:
    on-run-end: 
	- "SELECT ..."

In this lesson, we opt for the 2nd approach, specifically defining a BigQuery User Defined Function (UDF) in SQL to extract a brand name from a web URL.

  • Don’t worry if you don’t understand what regular expressions are, or how they work (they can get very very complex)
  • The main takeaway here is how you would use a macro to define a SQL snippet which we can then run before or after a dbt run very easily

Note, as this macro will now run every time you perform a dbt run / seed / snapshot! think carefully about how this will affect the developer experience.

File you need to create:

macros/macro_get_brand_name.sql

{#
	Creates a function in the target schema that can be used to extract the brand name
	from the events table web link column.

	For those not familiar with regex (regular expressions), they are basically a very powerful
	way to search within strings for any pattern of letters/numbers etc.

	In this example, we extract everything after /brand/ as all of the relevantweb links end
	with the brand name.

	e.g., take this web link: "/department/men/category/active/brand/columbia"
	Our regex match pattern, ".+/brand/(.+)", does the following:
	- ".+" matches ANYTHING 1 or more times. By itself, it would return the whole string!
	- "/brand/" matches exactly that in the web link.
		If we used ".+/brand/" as our match pattern, it would return "/department/men/category/active/brand/"
	- "(.+)" does 2 things:
		1. The brackets denote that this is the part we want to return
		2. ".+" will, again, match ANYTHING after "brand/"
#}
{% macro get_brand_name() %}
	CREATE OR REPLACE FUNCTION {{ target.schema }}.get_brand_name(web_link STRING)
	RETURNS STRING
	AS (
		REGEXP_EXTRACT(web_link, r'.+/brand/(.+)')
	)
{% endmacro %}

Code you’ll need to add to your dbt_project.yml file (anywhere):

on-run-start:
  - "{{ get_brand_name() }}"

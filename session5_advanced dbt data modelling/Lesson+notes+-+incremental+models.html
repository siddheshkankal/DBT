Lesson notes - incremental models

Incremental models are basically table models, but allow for data to be appended/updated rather than a full drop & replace each time.

They’re more complex as a result, but they can save you both money and time by selecting a far smaller portion of data. They’re worthwhile looking at once your tables get large enough that dropping & replacing them is a lengthy process.

A few things to call out:

  • Running dbt run --full-refresh (or dbt build --full-refresh) will drop and replace any incremental models (it’ll set is_incremental() to false)
  • The table expiration time we set in our dbt_project.yml won’t be updated when you run an incremental model, i.e. if you create an incremental table with 7 days to expiry - it will expire after 7 days (and force a full table recreation) unless you do a --full-refresh
    • You can override the table expiration in the model itself such as {{ config(hours_to_expiration=1000, ...) }}
  • If you don’t use the unique_key in your model’s config, then you may end up with duplicate data if, for the same unique key, you get data that’s retrospectively updated! dbt uses the unique_key to update data for the same key rather than appending a new row
  • We only touched briefly on the on_schema_change settings, and didn’t cover the incremental_trategy as that is more specific to the database provider. If you want to learn more - dbt’s documentation page on incremental models goes in to a lot of detail!
  • The {{ this }} function in dbt is basically a {{ ref('this model') }} without causing a circular dependency, read more here

Files you need to create:

models/staging/stg_ecommerce__events.sql

{{
	config(
		materialized='incremental',
		unique_key='event_id',
		on_schema_change='sync_all_columns'
	)
}}

WITH source AS (
	SELECT *

	FROM {{ source('thelook_ecommerce', 'events') }}
)

SELECT
	id AS event_id,
	user_id,
	sequence_number,
	session_id,
	created_at,
	ip_address,
	city,
	state,
	postal_code,
	browser,
	traffic_source,
	uri AS web_link,
	event_type

FROM source

{% if is_incremental() %}

WHERE created_at > (SELECT MAX(created_at) FROM {{ this }})

{% endif %}
